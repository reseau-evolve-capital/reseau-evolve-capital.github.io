# I18N-001 — Internationalisation de `apps/web` (FR par défaut)

> **Type** : follow-up (créé pendant la passe de correction de l'audit 2026-06-04).
> **Statut** : à planifier (NON traité dans la passe de correction de l'audit — décision actée).
> **Épopée** : Web / Infra. **Priorité** : moyenne (non bloquant).

## Contexte

L'audit 2026-06-02 (`docs/audits/AUDIT-2026-06-02.md`) et la mise à jour des garde-fous
(2026-06-04) ont acté : **le « FR-only » n'est plus une règle**. L'app membre `apps/web`
doit être **internationalisable**, avec le **français comme langue par défaut**.

Une string en anglais n'est pas un bug en soi ; le vrai défaut serait l'absence d'i18n ou
un écran non traduit/non brandé. Pendant la passe de correction, on a donc gardé le **copy
FR par défaut** et créé des chaînes custom FR (ex. `not-found.tsx`, error boundaries,
login split-panel) — sans mettre en place l'i18n réelle, qui est CE ticket.

## Objectif

Externaliser les chaînes de `apps/web` derrière une lib i18n, FR par défaut, ajout d'une
2ᵉ langue trivial.

## Périmètre

- Choisir la lib (ex. `next-intl` — compatible Next 16 App Router / RSC).
- Mettre en place le routing locale (par défaut `fr`, sans préfixe ou `/fr` selon stratégie SEO).
- Externaliser les chaînes des écrans : login/onboarding, dashboard, portfolio, contributions,
  admin, états (error/empty/404), composants `@evolve/ui` qui portent du copy.
- Formatters déjà localisés FR via `@evolve/utils` (`formatEUR`/`formatPct`/`formatDate`/
  `formatDateLong`) — à articuler avec la locale i18n (NBSP, fr-FR conservés par défaut).
- `not-found.tsx` + error boundaries : passer leurs chaînes par le système i18n (FR par défaut).

## Critères d'acceptation

- [ ] Strings de `apps/web` externalisées (catalogue de messages).
- [ ] Langue par défaut = FR ; l'app rend identique à aujourd'hui en FR.
- [ ] Ajout d'une 2ᵉ langue (ex. EN) = ajouter un fichier de messages, sans refactor.
- [ ] Pas de régression a11y / formatage monnaie & dates (rester via `@evolve/utils`).
- [ ] Tests : au moins un test de bascule de locale + non-régression du rendu FR.

## Notes

- Ne PAS internationaliser la **vitrine** `apps/vitrine` (déjà i18n FR/EN historique, ne pas refacto).
- Voir `docs/audits/AUDIT-2026-06-02.md` (F2, §I18N-001) et la mémoire projet `i18n-not-fr-only`.
