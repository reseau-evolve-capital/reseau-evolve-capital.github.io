# EDI-008 — Traduction automatique FR→EN du blog (assistée LLM, seed éditable)

> **Type** : follow-up / nouvelle feature CMS éditorial.
> **Statut** : à concevoir → prêt à implémenter (design validé avec l'owner, 2026-06-09).
> **Épopée** : E-EDI (éditorial & publication). **Priorité** : moyenne-haute (débloque la parité FR/EN du blog sans surcharger les rédacteurs).
> **Dépendances** : EDI-000 (Strapi extrait en `apps/cms`), EDI-001 (modèle `article` i18n + dynamic zone `corps`), déploiement CMS DigitalOcean (live `strapi.reseauevolvecapital.com`).
> **Branche cible** : `feat/edi-008-auto-translate` (depuis `main`).

---

## Contexte & motivation

Le club est aujourd'hui majoritairement francophone. Les rédacteurs écrivent les articles
(« La Quote-Part » + articles libres) **en français**, mais la vitrine est **bilingue FR/EN**
(`apps/vitrine/src/app/[locale]/blog/...`, build statique qui consomme `getAllArticles('fr'|'en')`).

Le content-type `article` est **déjà i18n** (`apps/cms/src/api/article/content-types/article/schema.json`) :
`title`, `slug`, `excerpt`, `content` (blocks natif), **`corps`** (dynamic zone de 9 blocs) et les champs
SEO sont `localized: true` ; `featuredImage`, `type`, `numeroEdition`, dates et auteur sont **non
localisés** (partagés entre locales → pas de ré-upload d'image par langue).

Aujourd'hui, produire la version EN d'un article = ressaisie manuelle complète. On veut **pré-remplir
automatiquement la locale EN à partir du FR** pour soulager les rédacteurs, **tout en laissant chaque
rédacteur retoucher librement le contenu EN par la suite**.

**Contrainte de conception décisive** : « laisser les rédacteurs modifier l'EN eux-mêmes plus tard »
**interdit le modèle de synchronisation continue** (où chaque sauvegarde FR réécrase l'EN). On retient
un modèle **« seed-once-then-editable »** : la traduction est une **action explicite** qui amorce un
brouillon EN ; elle n'écrase jamais silencieusement une édition manuelle.

## Décisions structurantes (validées avec l'owner)

| #   | Décision | Justification |
| --- | --- | --- |
| 1   | **Plugin `strapi-llm-translator`** (grenzbotin), pas la trad native Strapi AI | La trad native (v5.30+) est **Cloud-only / plan Growth** et fonctionne en **sync-on-save** → incompatible avec notre auto-hébergement DO **et** avec le besoin « éditer l'EN après ». Le plugin requiert `>=5.12.3` → OK sur notre **5.12.6**, gère **string/text/richtext + JSON/blocks + dynamic zones** (récursif). |
| 2   | **Moteur = OpenAI** (compatible OpenAI), modèle `gpt-4o` par défaut | Choix owner. Le plugin est LLM-agnostique (tout endpoint compatible OpenAI). Un LLM préserve mieux le **ton éditorial** de « La Quote-Part » et le **vocabulaire finance** qu'une trad littérale. |
| 3   | **Déclenchement manuel par bouton** (Content Manager, locale EN), **jamais on-save** | Respecte le modèle seed-then-edit : retraduire est un choix explicite, pas un écrasement automatique. |
| 4   | **EN amorcé en brouillon** (draftAndPublish), publié après relecture | Le rédacteur relit, corrige les passages qui comptent, puis publie. La vitrine ne build que les locales **publiées**. |
| 5   | **`slug` exclu de la traduction** (copié verbatim du FR) | `slug` est `localized: true` **et** `unique`. On garde le slug FR pour l'EN (routing simple, pas de collision). SEO EN du slug = amélioration différée (hors périmètre). |
| 6   | **Glossaire / consigne de marque** dans la config du plugin | « La Quote-Part », « quote-part », noms de produits et raison sociale **non traduits** ; registre finance ; ton éditorial conservé. |
| 7   | **Zéro changement côté `apps/vitrine`** | La consommation (`getAllArticles`, renderers blocks/dynamic zone) est déjà locale-aware. Toute la feature vit dans `apps/cms`. |

## Périmètre fonctionnel

### Workflow rédacteur (le cœur du ticket)

1. Rédiger l'article **en FR** comme aujourd'hui (Content Manager).
2. Basculer sur la **locale EN** de l'article → cliquer le **bouton de traduction** du plugin
   (« Translate from another locale » → source = French).
3. Le plugin parcourt **tous les champs localisés** — `title`, `excerpt`, `content` (blocks),
   `corps` (dynamic zone, récursivement à travers ses 9 composants), `SEOMetaTitle`,
   `SEOMetaDescription` — et **remplit le brouillon EN**. `slug` est **copié, pas traduit** (décision #5).
   `featuredImage` / dates / auteur restent partagés (non localisés).
4. Le rédacteur **relit, corrige, publie** la locale EN.
5. Au prochain build vitrine, le blog EN est à jour. **Aucune intervention dev.**

### Comportement attendu

| Cas | Attendu |
| --- | --- |
| Article FR sans EN existant | Le bouton crée/amorce la locale EN (même `documentId` que le FR — voir vérif runtime). |
| Article EN déjà édité manuellement | Retraduire **réécrit** le contenu EN → action explicite uniquement, jamais déclenchée par une sauvegarde FR. Comportement à **documenter** pour les rédacteurs (« retraduire écrase vos retouches EN »). |
| Champ `slug` | Identique au FR (non traduit). |
| Bloc non textuel (image, séparateur, le-chiffre) | Champs texte traduits ; média/valeurs structurelles préservés. |
| Échec API LLM (quota, réseau) | Message d'erreur clair dans l'admin, **aucun écrasement partiel** laissant l'EN dans un état incohérent (à vérifier au test). |

## Solution technique

### 1. Dépendance

`apps/cms/package.json` : ajouter `strapi-llm-translator` (dernière version compatible v5 ;
testée v5.12.x / v5.15.x). Installer via `yarn` (app autonome, yarn classic — cf. `apps/cms/CLAUDE.md`),
puis `yarn build` (rebuild de l'admin pour embarquer le plugin).

### 2. Configuration

Le plugin se configure **en partie par variables d'environnement** et **en partie via la page de
configuration du plugin dans l'admin** (exclusions de champs, consigne/glossaire, modèle). Variables
d'environnement lues par le plugin :

| Variable | Rôle | Valeur |
| --- | --- | --- |
| `LLM_TRANSLATOR_LLM_API_KEY` | **Clé API du LLM** | ⚠ **C'est ici qu'on met la clé OpenAI.** (Le plugin n'utilise pas le nom `OPENAI_API_KEY`.) |
| `STRAPI_ADMIN_LLM_TRANSLATOR_LLM_BASE_URL` | Endpoint | Défaut OpenAI → laisser vide sauf proxy/Azure |
| `STRAPI_ADMIN_LLM_TRANSLATOR_LLM_MODEL` | Modèle | Défaut `gpt-4o` (override possible, ex. `gpt-4o-mini` pour réduire le coût) |
| `STRAPI_ADMIN_LLM_TRANSLATOR_AZURE_API_VERSION` | Azure uniquement | Non utilisé (OpenAI direct) |

> **Mapping clé** : la valeur de `LLM_TRANSLATOR_LLM_API_KEY` = la **clé OpenAI** fournie par l'owner.
> Le nom de la variable reste celui imposé par le plugin.

**Réglages dans la page de config admin du plugin** (à faire une fois après déploiement) :
- **Exclure `slug`** de la traduction (décision #5).
- **Consigne système / glossaire** (décision #6) : ne pas traduire « La Quote-Part », « quote-part »,
  « Réseau Evolve Capital », noms de produits ; conserver le registre finance et le ton éditorial.
- Vérifier que `content`, `corps`, `excerpt`, `title`, `SEOMeta*` sont bien inclus.

### 3. Secrets & déploiement

- **Local** (`apps/cms/.env`) : ajouter `LLM_TRANSLATOR_LLM_API_KEY=<clé OpenAI>` (pour tester). Ne pas
  écraser le `.env` réel (cf. gotcha `apps/cms/CLAUDE.md` — ne pas lancer `make strapi-env`).
- **Prod (droplet DO)** : ajouter `LLM_TRANSLATOR_LLM_API_KEY` au `.env.production` du compose
  `-p strapi`, et au modèle `apps/cms/.env.production.example` (sans la valeur). Mettre à jour le
  runbook `docs/infra/RUNBOOK-cms-digitalocean.md` (section secrets).
- **Image CMS** : la CI `.github/workflows/deploy-cms.yml` build déjà l'image GHCR sur push `main`
  (`paths: apps/cms/**`). L'ajout du plugin sera embarqué automatiquement au prochain build. Aucune
  étape CI nouvelle requise (vérifier seulement que `yarn build` passe avec le plugin).

### 4. Permissions admin

Accorder au(x) rôle(s) éditeur(s) l'accès à l'action du plugin (settings → roles), sinon le bouton
de traduction n'apparaît pas dans le Content Manager.

## Vérifications runtime obligatoires (avant prod)

> Le « gate vert » statique ne suffit pas : la preuve, c'est le comportement observé dans l'admin.

1. **Liaison de locale (`documentId`)** — quirk connu Strapi 5 (Document Service, issue #24445) :
   une nouvelle locale peut se voir attribuer un **nouveau `documentId`** au lieu d'être liée à
   l'original. **Tester sur un article jetable** : traduire FR→EN, puis confirmer que l'entrée EN
   **partage le `documentId`** du FR (sinon slug/relations se découplent et le blog casse). Si le
   plugin ne lie pas correctement → escalader (downgrade du périmètre vers création manuelle de la
   locale EN puis trad, ou patch).
2. **`slug` unique** — vérifier qu'EN reprend le slug FR sans violer la contrainte `unique`
   (en i18n, l'unicité est par locale → OK attendu, mais à confirmer runtime).
3. **Dynamic zone `corps`** — traduire un article contenant **chacun des 9 types de blocs** et
   vérifier que la structure est préservée (médias, `le-chiffre`, `separateur`, `cta`) et que seuls
   les champs texte sont traduits.
4. **Rendu vitrine EN** — après publication EN, lancer un build vitrine local et vérifier le rendu
   `/en/blog/[slug]` (light + dark) : pas de FR résiduel, pas de bloc cassé.
5. **Idempotence / non-écrasement FR** — sauvegarder le FR après coup et confirmer que l'EN n'est
   **pas** modifié (preuve que ce n'est pas du sync-on-save).

## Risques & limites connus

- **Coût LLM** : volume faible (cadence newsletter) → négligeable. `gpt-4o-mini` possible si on veut
  réduire encore, au prix d'un peu de finesse de ton.
- **Quirk `documentId` Strapi 5** (issue #24445) : risque principal — d'où la vérif #1 bloquante.
- **Confidentialité** : le contenu éditorial transite par l'API OpenAI. Contenu = blog **public** →
  acceptable. Ne pas étendre ce flux à du contenu membre/privé sans réévaluation.
- **Single-instance DO** : pas de souci de concurrence (un seul process Strapi).
- **Qualité EN** : seed, pas vérité finale. La relecture humaine reste l'étape de publication.

### Workflow réel (≠ « un seul clic »)

Le bouton **« Translate with AI »** traduit le contenu de **la locale courante en place** (il ne va pas
chercher la source tout seul). Workflow correct, désormais documenté pour les rédacteurs :
**locale EN → « Fill in from another locale » (French) → Save → « Translate with AI » → relire →
remettre le slug FR → Publier.** (Le bouton reste « Translation completed » jusqu'au rechargement de la fiche.)
